# 訂單郵件萃取 Skill

這是一個用於 Codex 的訂單整理 Skill，角色定位為「訂單管理與資料分析秘書」。它可以從客戶零散的電子郵件、聊天訊息或非正式訂購內容中，萃取出標準化的訂單資訊。

## 功能特色

- 過濾寒暄、簽名檔與無關討論。
- 提取客戶姓名、電話、地址、產品類別、設計主題、技術規格與交付資訊。
- 將尺寸統一為 `mm`，數量統一為 `pcs`，日期統一為 `YYYY/MM/DD`。
- 對缺漏或不明確的資訊標註「待確認」。
- 使用繁體中文 bullet points 輸出。
- 根據訂單需求提供 1-2 個專業建議。

## 使用方式

在 Codex 中輸入：

```text
Use $extract-order-emails 幫我整理下面這封客戶郵件：

嗨您好，我想做一隻橘貓公仔，大概高 8 公分，坐姿，要有紅色項圈。
數量先做 2 個，希望 2026/4/10 前收到，是要送朋友的生日禮物。
電話 0912-345-678，寄到台北市信義區...
```

## 輸出範例

```text
【訂單提取結果】

- 客戶姓名：待確認
- 聯絡電話：0912-345-678
- 收件地址／取貨方式：台北市信義區...
- 產品類別：客製化公仔
- 設計主題：橘貓公仔，坐姿，紅色項圈
- 技術規格：高 80mm；顏色要求：橘貓配色、紅色項圈
- 交付資訊：2 pcs；希望收件日期：2026/04/10
- 備註事項：送朋友的生日禮物
- 待補資訊：客戶姓名、完整收件地址

【專業建議】

- 建議使用樹脂列印以提升貓咪毛色、坐姿輪廓與項圈細節。
- 若作為禮物，建議確認是否需要加購透明展示盒或禮盒包裝。
```

## GitHub Pages

本 repository 也包含 `index.html`，可以作為 GitHub Pages 展示頁。

啟用方式：

1. 進入 GitHub repository 的 `Settings`。
2. 點選 `Pages`。
3. Source 選擇 `Deploy from a branch`。
4. Branch 選擇 `main`，資料夾選擇 `/root`。
5. 儲存後等待 GitHub 產生公開網址。
